# Reverse Spec Code Analysis

## 目的

コードとテストから、利用者が観測できる挙動を抽出する。

## 手順

1. CLI 層を読む
   - コマンド名
   - 引数とオプション
   - エラーハンドリング
2. Application 層を読む
   - ユースケースの入力と出力
   - ドライランや分岐条件
3. Infrastructure 層を読む
   - ファイル形式判定
   - 入出力仕様
   - サポート拡張子
4. テストを読む
   - 実装に対する保証範囲
   - エッジケース
5. README と照合する
   - コマンド例
   - 利用想定
6. 根拠をメモする
   - 仕様ごとに対象ファイルを控える

## 抽出メモの観点

- 前提条件
- 入力形式
- 成功時出力
- 失敗時出力
- 変更有無
- 並び順や形式などの再現性条件

## 出力メモの例

```text
カテゴリ: scan
機能名: 出力先未指定時の標準出力
内容: output 未指定時は CSV を STDOUT に出力する
備考: ヘッダー行を含む
根拠: src/.../cli/main.py, tests/test_cli_scan.py
```
