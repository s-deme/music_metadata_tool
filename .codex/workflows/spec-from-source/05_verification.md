# Reverse Spec Verification

## 目的

CSV と生成ドキュメントが実装と矛盾していないかを検証する。

## 手順

1. CSV の各行について根拠ファイルを再確認する
2. `status=needs-review` の行を重点的に見直す
3. README と食い違う項目を洗い出す
4. 仕様として重要なエラー条件が漏れていないか確認する
5. CSV から Markdown に整形する
6. 必要に応じて HTML に変換する

## 検証観点

- コマンドが不足していないか
- オプションの条件付き挙動が抜けていないか
- 異常系が正常系に埋もれていないか
- `config.json` の影響が反映されているか
- テスト未保証の項目が区別されているか

## 生成物の品質基準

- 仕様の各段落が CSV の行に対応している
- 実装根拠に辿れる
- 利用者が手順と注意点を理解できる
